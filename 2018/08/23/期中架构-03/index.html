rsync守护进程应用场景

1）定时任务方式+rsync进行数据备份
备份的数据：内部人员产生的数据
2）实时同步方式+rsync进行数据备份
备份的数据：外部人员产生的数据

rsync守护进程企业实际应用

全局配置：所有模块以上配置的信息
局部配置：在相应模块下的配置

守护进程来创建备份目录

rsync -avz  /etc/hosts  oldgirl@172.16.1.41::backup/dev/ --password-file=/etc/rsync.password
ps: 无法一次性创建多级目录(没有mkdir -p功能)

守护进程无差异同步配置(在企业中慎用)

说明: 我有的你也有,我没有的你也不能拥有
# rsync -avz --delete /oldboy_dir/  oldgirl@172.16.1.41::backup --password-file=/etc/rsync.password

说明: 实现存储服务器和备份服务器数据信息高度一致
补充: 无差异同步,可以更加快速的清空目录数据信息或者文件内容信息

守护进程的列表功能配置

list = true
作用: 让客户端可以通过命令获知服务端的全部模块信息
[root@nfs01 ~]# rsync oldgirl@172.16.1.41::
backup             "backup dir by oldboy"
mysql 
建议: 将list参数指定为false, 提升架构安全性

kill:

killall(建议)

pkill(模糊的杀手-慎用)    
pkill 进程名
进程杀死后不会有提示信息
