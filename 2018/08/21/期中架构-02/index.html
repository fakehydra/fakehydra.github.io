期中架构-02 rsync

rsync备份服务软件介绍

rsync是一款开源的、快速的、多功能、多功能的、可实现全量及增量的本地或远程数据备份
增量备份：只同步变化的数据信息，主要利用了quick check算法
同步原理：
 1) 建立好网络连接
 2) 先进行数据信息比对
 3) 只将变化的数据进行增量备份

rsync软件作用说明：（1v4）

本地数据备份：cp

[root@backup ~]# cp -a /etc/hosts /tmp/hosts_cp
[root@backup ~]# ll /tmp/hosts_cp
-rw-r--r--. 1 root root 405 Aug 20 15:28 /tmp/hosts_cp
[root@backup ~]# rsync -a /etc/hosts /tmp/hosts_rsync
[root@backup ~]# ll /tmp/hosts_rsync
-rw-r--r-- 1 root root 405 Aug 20 15:28 /tmp/hosts_rsync

PS：同步目录的时候目录后面有/（null/）,表示同步目录下的文件；没有/（/root/null）表示同步目录以及目录下的文件

[root@backup ~]# rsync -a null/ /tmp/
[root@backup ~]# ll /tmp/
total 16
-rw-r--r--  1 root root   0 Aug 21 09:30 1.txt
-rw-r--r--. 1 root root 405 Aug 20 15:28 hosts_cp
-rw-r--r--  1 root root 405 Aug 20 15:28 hosts_rsync
-rw-r--r--  1 root root  28 Aug 21 09:27 network_rsync1
-rw-r--r--  1 root root  28 Aug 20 15:46 network_scp
-rw-------. 1 root root   0 Aug 18 15:08 yum.log
[root@backup ~]# rsync -a /root/null /tmp/
[root@backup ~]# ll /tmp/

-rw-r--r--  1 root root    0 Aug 21 09:30 1.txt
-rw-r--r--. 1 root root  405 Aug 20 15:28 hosts_cp
-rw-r--r--  1 root root  405 Aug 20 15:28 hosts_rsync

-rw-r--r--  1 root root   28 Aug 20 15:46 network_scp
drwxr-xr-x  2 root root 4096 Aug 21 09:30 null

远程数据备份：scp
[root@nfs ~]# scp -rp /etc/sysconfig/network backup:/tmp/network_scp
[root@nfs ~]# rsync -rp /etc/sysconfig/network backup:/tmp/network_rsync1

[root@backup ~]# ll /tmp/

[root@backup ~]# rsync -a --delete /root/null/ /tmp/
[root@backup ~]# ll /tmp/

[root@backup ~]#

rsync特点说明：

1) 支持拷贝普通文件与特殊文件如链接文件，设备等。
2) 可以有排除指定文件或目录同步的功能，相当于打包命令tar的排除功能。
  #tar zcvf backup_1.tar.gz  /opt/data  -exclude=oldboy    
  说明：在打包/opt/data时就排除了oldboy命名的目录和文件。
3) 可以做到保持原文件或目录的权限、时间、软硬链接、属主、组等所有属性均不改变-p。
4) 可实现增量同步，既只同步发生变化的数据，因此数据传输效率很高（tar -N）。
  # 将备份/home 目录自 2008-01-29 以来修改过的文件
  # tar -N 2008-01-29 -zcvf /backups/inc-backup_$(date +%F).tar.gz /home
  # 将备份 /home 目录昨天以来修改过的文件
  # tar -N $(date -d yesterday "+%F") -zcvf /backups/inc-backup_$(date +%F).tar.g home

  # 添加文件到已经打包的文件
  # tar -rf all.tar *.gif
  说明：这条命令是将所有.gif的文件增加到all.tar的包里面去。-r是表示增加文件的意思。
5) 可以使用rcp,rsh,ssh等方式来配合进行隧道加密传输文件（rsync本身不对数据加密）
6) 可以通过socket(进程方式)传输文件和数据（服务端和客户端）*****。重点掌握
7) 支持匿名的或认证（无需系统用户）的进程模式传输，可实现方便安全的进行数据备份及镜像。

rsync软件的工作方式（如何应用）

本地备份数据方式

     1) 本地备份数据方式(cp命令)
Local:  rsync [OPTION...] SRC... [DEST]
rsync              --- 数据备份命令
[OPTION...]        --- 命令参数信息
src                --- 要进行同步备份的源文件或目录信息
[DEST]             --- 将数据备份同步到本地系统中的什么路径下

2) 远程备份数据方式
Access via remote shell:
Pull: rsync [OPTION...] [USER@]HOST:SRC... [DEST]
rsync              --- 数据备份命令
[OPTION...]        --- 命令参数信息
[USER@]            --- 以什么用户身份进行数据的拉去    
HOST:SRC           --- 从哪个远程主机上,将指定的数据进行备份同步
[DEST]            --- 将远程主机拉去过来的数据保存到本地什么路径中

Push: rsync [OPTION...] SRC... [USER@]HOST:DEST
rsync              --- 数据备份命令
[OPTION...]        --- 命令参数信息
[USER@]            --- 以什么用户身份进行数据的拉去    
SRC                --- 指定本地要推送备份的数据信息
[USER@]HOST:DEST   --- 以什么用户身份,将数据信息推送到指定主机的相应目录中    

参照服务器: 在什么服务器执行操作命令,对应的服务器就是参照服务器
推: 在参照服务器上执行命令, 将本地数据怼到远程主机上
拉: 在参照服务器上执行命令, 将远程数据恏到本地主机上

rsync守护进程服务部署方式

服务端：
1）确认rsync是否安装
[root@backup ~]# rpm -qa rsync
rsync-3.0.6-12.el6.x86_64
2）编写配置文件


3）创建用户
[root@backup ~]# useradd -M -s /sbin/nologin rsync 
[root@backup ~]# id rsync
uid=501(rsync) gid=501(rsync) groups=501(rsync)
4）创建目录
[root@backup ~]# mkdir /backup -p
[root@backup ~]# chown -R rsync.rsync /backup
5）创建认证用户密码文件
[root@backup ~]# echo "rsync_backup:123456" >>/etc/rsyncd.password
[root@backup ~]# cat /etc/rsyncd.password
rsync_backup:123456
[root@backup ~]# chmod 600 /etc/rsyncd.password
6）启动服务的守护进程
[root@backup ~]# rsync --daemon
[root@backup ~]# ps -ef |grep rsync
root      26024      1  0 11:34 ?        00:00:00 rsync --daemon
root      26026  25304  0 11:34 pts/0    00:00:00 grep rsync

PS：rsync服务端口号为873
