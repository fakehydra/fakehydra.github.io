第二十五天–磁盘管理（下）

MBR引导 —分区概念知识相关

引导系统启动记录；系统磁盘一些分区信息
记录在磁盘的0磁头、0磁道、1扇区

企业案例：系统引导启动不了？

1.保存好引导记录：0磁头、0磁道、1扇区的数据
2.恢复。

磁盘分区类型：

主分区（primary）:总共最多四个（MBR）；每个16字节，共64字节
扩展分区（extended）：可以划分多个逻辑分区，一个分区表中只能有一个扩展分区；扩展分区不能直接使用
逻辑分区（logical）：细化扩展分区，用于存储数据

磁盘分区实践：

第二步：让系统识别加载生成的分区表信息：

[root@jinc ~]# partprobe /dev/sdb

第三步：对分区进行格式化

[root@jinc ~]# mkfs.ext4 /dev/sdb5

说明：扩展分区无法进行格式化操作

第四步：禁止磁盘自检操作

[root@jinc ~]# tune2fs -c 0 -i 0 /dev/sdb5

第五步：挂载

[root@jinc ~]# mount /dev/sdb5 /mnt/

b：磁盘容量大于等于2TB

利用parted命令进行分区，分区的引导记录---GPT

系统分区引导记录分类：

MBR：引导磁盘大小小于2TB --msdos
GPT：引导磁盘大小大于2TB

创建大于2TB的磁盘

parted命令分区参数说明

                                         指定分区的引导记录信息

                                         partitions, or a particular partition
                                         显示出分区信息

                                         删除分区

                                         退出分区操作（默认保存）

总结：fdisk命令和parted命令区别

01.分区的磁盘大小限定（2TB）
02.分区方式上（分区过程放在内存中-fdisk；分区过程实时生效）
    （fdisk只能在分区界面进行分区--parted可以在命令行完成）

企业案例-01

Java环境内存不够用了，大量使用swap分区

解决方式：

01.创建一个用于给swap分区的文件
dd if=/dev/zero of=/tmp/100m bs=1M count=100
02.标记创建好的文件作为swap分区使用
[root@jinc1 ~]# mkswap /tmp/100m 
mkswap: /tmp/100m: warning: don't erase bootbits sectors
        on whole disk. Use -f to force.
Setting up swapspace version 1, size = 73724 KiB
no label, UUID=041f19e5-ad77-405e-88a6-7a3bb172c446
[root@jinc1 ~]# file /tmp/100m 
/tmp/100m: Linux/i386 swap file (new style) 1 (4K pages) size 18431 pages
[root@jinc1 ~]#
03.进行加载文件被swap分区使用
[root@jinc1 ~]# swapon /tmp/100m 
04.查看swap交换分区信息
[root@jinc1 ~]# swapon -s
Filename                Type        Size    Used    Priority
/dev/sda2                               partition    1535996    25276    -1
/tmp/100m                               file        73724    0    -2

取消掉不需要的

[root@jinc1 ~]# swapoff /tmp/100m
