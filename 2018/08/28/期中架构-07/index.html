实时同步服务回顾

   inotify软件
inotifywait命令常见参数:
-r   --- 实现监控目录中子目录里面数据变化
-m   --- 实现持续监控
-q   --- 让输出的信息显示的少一些
--timefmt  -- 指定时间格式信息
--format   -- 指定输出信息的格式
-e   --- 指定监控的事件信息,默认如果不进行指定,表示监控所有事件

inotify软件部署条件
max_user_watches:    设置inotifywait或inotifywatch命令可以监视的文件数量（单进程）
cat /proc/sys/fs/inotify/max_user_watches 
50000000

max_user_instances:    设置每个用户可以运行的inotifywait或inotifywatch命令的进程数
说明: 指定可以监控多少个目录
# cat /proc/sys/fs/inotify/max_user_instances 
128

max_queued_events:    设置inotify实例事件（event）队列可容纳的事件数量
cat /proc/sys/fs/inotify/max_queued_events 
327679

永久优化inotify软件方法

echo 'echo 6000000000 >/proc/sys/fs/inotify/max_user_watches' >>/etc/rc.local 
echo 'echo 218 >/proc/sys/fs/inotify/max_user_instances' >>/etc/rc.local 
echo 'echo 6500000000 >/proc/sys/fs/inotify/max_queued_events' >>/etc/rc.local 
说明: 以上三个文件也是inotify软件优化点

高并发数据实时同步方案小结：

1)    inotify（sersync）+rsync，是文件级别的。
2)    drbd存储底层级别，基于block块同步，但是备节点数据不可用
3)    第三方软件的同步功能：mysql同步(主从同步)，oracle(集群)，mongodb。
4)    程序双写，直接写两台服务器
5)    NFS集群（1、4、5方案整合）（双写主存储，备存储用inotify/sersync+rsync），备没有找主解决延迟问题

ssh远程连接原理

1) 执行命令发出远程连接请求       客户端---服务端
   ssh 10.0.0.31
2) 回复请求信息,发送连接确认请求  服务端---客户端
   The authenticity of host '10.0.0.31 (10.0.0.31)' can't be established.
   RSA key fingerprint is 0e:49:3c:b7:5f:4c:45:43:b8:de:f3:0b:0c:2a:e7:d1.
   Are you sure you want to continue connecting (yes/no)? 
3) 进行确认,表示想进行远程登录    客户端---服务端
   Are you sure you want to continue connecting (yes/no)? yes
   ll ~/.ssh/known_hosts 
   -rw-r--r-- 1 root root 391 Dec 12 07:37 /root/.ssh/known_hosts   --- 输入yes后,文件内容增加
   说明: 此文件中保存公钥信息
4) 发送公钥信息(锁头),询问登录密码信息          服务端---客户端
    cat /etc/ssh/ssh_host_rsa_key.pub
5) 客户端接收公钥,并进行保存,输入登录密码信息   客户端---服务端
   ~/.ssh/known_hosts
6) 服务端确认密码信息,建立远程连接               服务端---客户端

SSH服务远程连接认证类型

基于密码的认证方式
基于秘钥的认证方式
1. 管理端: 创建私钥和公钥(创建秘钥对)
2. 管理端: 分发公钥信息(第一次分发公钥的时,仍旧采用基于密码建立连接)
3. 受控端: 接收到公钥文件信息,进行保存
4. 管理端: 发送建立连接请求
5. 受控端: 发送公钥质询信息
6. 管理端: 响应公钥质询信息
7. 受控端: 接收响应信息,确认无误,建立连接
