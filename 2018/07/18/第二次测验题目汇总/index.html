第二次测验

第一题：如何过滤出已知当前目录下oldboy中的所有一级目录

(提示:不包含oldboy目录下面目录的子目录及隐藏目录，即只能是第一级目录)？五种方法

创建模拟环境

cd ~
mkdir oldboy -p
cd oldboy/
mkdir ext/oldboy test xiaodong xiaofan xingfujie -p
touch jeacen oldboy wodi.gz yingsui.gz
LANG=en

方法一：利用tree命令

tree -Ld 1

方法二：利用find命令

find /root/oldboy/ -type d -maxdepth 1 （绝对路径）
find ./ -maxdepth 1 -type d （相对路径）

方法三：利用三剑客命令

ls -l /root/oldboy/|grep "^d"  --以d开头
ll -F /root/oldboy/|grep "/$"  --以/结尾

ls -l /root/oldboy/|sed -n "/^d/p" --以d开头
ll -F /root/oldboy/|sed -n '/\//p' --以/结尾
ll -F /root/oldboy/|sed -n '/[/]/p' --以/结尾

ls -l /root/oldboy/|awk "/^d/" --以d开头
ll -F /root/oldboy/|awk '/\//' --以/结尾

方法四：利用硬连接

ls -l /root/oldboy/|awk '$2>1'
ls -l /root/oldboy/|awk '$2!=1'

方法五：利用*

ls -ld */

第二题：如何进行目录之间的快速切换

[root@jinc oldboy]# cd /proc/sys/net/ipv4
[root@jinc ipv4]# cd /tmp/
[root@jinc tmp]# cd -
/proc/sys/net/ipv4
[root@jinc ipv4]#

说明

-  ：还原切换以前的目录信息；等价于$OLDPWD
$OLDPWD ：记录保存切换之前的所在路径信息

cd命令用法总结

cd - :切换到上一次所在的目录信息
   ..:切换到当前目录的上一级目录
   . ：切换到当前所在的目录（一般用在移动到当前目录时使用）
   ~ ：当前登录系统的家目录信息

第三题：一个目录中有很多文件（ls查-看时好多屏），想最快速度查看到最近更新的文件。如何看？

ls -t 
利用-t参数可以按照时间顺序排序显示文件内容（由新到旧）
ls -rt 
利用-r参数，可以按照时间信息反向排序

第四题：已知nginx服务的访问日志按天记录在服务器本地目录/app/logs下，由于磁盘空间紧张，现在要求只能保留最近7天访问日志！

环境创建

mkdir /app/logs -p

解题

由于还没搭建Nginx(虽然我自己搭好了)，所以选择是创建一个环境。因为是要删除七天前的，所以我们先得把时间修改一下，然后创建相应的文件。
第一里程：修改系统时间
[root@jinc1 ~]# date -s "20150105"
 Mon Jan  5 00:00:00 CST 2015
 [root@jinc1 ~]# date
 Mon Jan  5 00:00:03 CST
第二里程：按天创造文件
由于我们要的是删除七天前的文件，所以我们所创建的文件至少要有八个以上，这样的话一个一个创建就太
慢了些，在这里，我们写个脚本来完成。

循环脚本：
for 循环的变量 in 范围 ；do 循环做的事 ；done

所以我们的脚本应该写成：
 for time in {01..20} ;do date -s "2015-01-$time";touch /app/logs/access_log."$date";done
 环境搭建完成

解题方法

方法一

解题核心：找出七天前的数据，删除

第一步：找出七天前的数据

find /app/logs -type f -mtime +7

-time :按天找寻数据信息
01.找寻几天前的   +x
02.找寻几天后的   -x
03.找寻第几天的    x

第二步：删除找到的东西

方法一：[root@jinc1 logs]# find /app/logs -type f -mtime +7|xargs rm  

方法二：[root@jinc1 logs]# find /app/logs -type f -mtime +7 -exec rm {} \;

方法三：[root@jinc1 logs]# find /app/logs -type f -mtime +7 -delete

方法四：[root@jinc1 logs]#\rm $(find /app/logs -type f -mtime +7)

第六题：查找/oldboy下所有以log结尾的大于1M的文件复制到/tmp目录

创建模拟环境

[root@jinc1 oldboy]# cat /etc/services /etc/services >>old.txt 
[root@jinc1 oldboy]# ll old.txt -h
-rw-r--r-- 1 root root 1.3M Jul 17 10:26 old.txt
说明：cat用于合并多个文件

解题思路

第一步：找出大与1M的文件
命令：find /oldboy -type f -name "*.txt" -size +1M
[root@jinc1 oldboy]# find ./ -type f -name "*.txt" -size +1M
   ./old03.txt
   ./old02.txt
   ./old.txt
   ./old01.txt
第二步：将找出的文件复制到/tmp目录
方法一：[root@jinc1 oldboy]# find ./ -type f -name "*.txt" -size +1M|xargs cp -t /tmp
方法二：[root@jinc1 oldboy]# find ./ -type f -name "*.txt" -size +1M|xargs -i cp {} /tmp
方法三：[root@jinc1 oldboy]# find ./ -type f -name "*.txt" -size +1M|xargs -I {} cp {} /tmp

第七题：打印配置文件nginx.conf内容的行号及内容，该如何做？

模拟环境创建

[root@jinc1 oldboy]# echo stu{01..20} |xargs -n1 > nginx.conf

解题方法

方法一：vim编辑。。就不多说了。。。一般人都会
方法二：利用cat命令
[root@jinc1 oldboy]# cat -n nginx.conf 
方法三：利用三剑客命令：
01：grep 
 [root@jinc1 oldboy]# grep -n "." nginx.conf 
02.sed
 [root@jinc1 oldboy]# sed '=' nginx |xargs -n2
03.awk
 [root@jinc1 oldboy]# awk '{print NR,$1}' nginx.conf

第九题：装完系统后，希望让网络文件共享服务NFS(iptables)，仅在3级别上开机自启动，该如何做？

所有级别实现防火墙开机启动

设置方法：

[root@jinc ~]# chkconfig iptables on  --开启
[root@jinc ~]# chkconfig |grep ipt  --查看

设置服务在指定级别开机自启

设置方法：

[root@jinc ~]# chkconfig --level 3 iptables on
[root@jinc ~]# chkconfig |grep ipt

第十一题：如何进行系统数据 压缩 打包/etc/目录

a.请用tar打包/etc整个目录（打包及压缩）。

tar zcvf 箱子  要打包的数据信息/cattar

tar zcvf /tmp/oldboy.tar.gz /etc/

    z   --表示压缩数据的方式  gzip
    c   --创建需要压缩的数据包 create
    v   --详细显示压缩过程（可不用）
    f   --指定生成压缩文件路径信息

检查压缩包数据信息方法：tar tf  压缩包
                          t :检查压缩包数据信息

b.请用tar打包/etc整个目录（打包及压缩，但需要排除/etc/services文件）。

[root@jinc etc]# tar zcf /tmp/oldgirl.tar.gz /etc/ --exclude=/etc/services 

 --exclude=要排除的数据信息    排除=后面的数据信息

c.请把a点命令的压缩包，解压到/tmp指定目录下（最好只用tar命令实现）。

 tar zxvf oldboy.tar.gz -C /tmp
x   --解压提取数据信息
说明：解压时，如果解压的数据时间信息不准确会有报错

第十二题：如何进行文件字符信息过滤

环境创建

mkdir /oldboy -p ;echo "i am oldboy,myqq is 664341340" >/oldboy/oldboy.txt

解题

第一种方法
sed 's#i am ##g' oldboy/oldboy.txt|sed "s#myqq is##g"
第二种方法：利用awk 命令
awk -F "[ ,]" '{print $3,$6}' /oldboy/oldboy.txt 

awk -F "[ ,]" '{print $3,$NF}' /oldboy/oldboy.txt 

awk 'BEGIN{FS="[ ,]"}{print $3,$6}' oldboy.txt
第三种方法：利用cut命令
sed 's#,# #g' /oldboy/oldboy.txt|cut -d " " -f3,6

第十三题：如何详细查看文件行数信息

第一种方法：

cat -n line.txt|tail -1|awk '{print $1}'

第二种方法：

wc line.txt
wc -l kine.txt

第十四题：过滤出/etc/services 文件包含3306或1521两数字所在的行的内容。

方法一：

grep -E "3306|1521" /etc/services

或者 egrep "3306|1521" /etc/services

方法二：

sed -n "/3306\|1521/p" /etc/services

或者 sed -rn "/3306|1521/p" /etc/services

方法三：

awk "/3306|1521/" /etc/services
