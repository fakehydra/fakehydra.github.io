linux运维学习 第三天

Share

服务器操作系统网络配置

如何实现网络远程管理虚拟主机（Linux系统）

两个配置步骤

步骤一
a)虚拟软件的网络相关配置
编辑–虚拟网络编辑器
网络模式：NAT模式
主机连接适配器 勾选
是否开启DHCP功能 勾选
子网：10.0.0.0 子网掩码：255.255.255.0

b）ip地址是什么鬼：你家的门牌号（地址）
可以自己配置ip地址：静态ip
别人分配ip地址：动态ip地址（DHCP–Dynamic Host Configuration Protocol)

c)虚拟主机中网卡设置
连接–勾选
启动时连接–勾选
网卡模式–勾选

步骤二
虚拟主机中操作系统网络配置

setup–网络设置–设备配置–eth0(eth1 eth2)
DNS:域名解析（将域名转换为ip地址）

开启DHCP功能
配置好DNS服务器地址
将on boot 勾选
ok–save–save&quit–quit–q

*linux系统中所有的配置想要生效–重启服务（网络/网卡服务）

重启网卡eth0命令

ifdown eth0–关闭网卡 eth0
ifup eth0–开启网卡

*linux 系统中当所有配置完毕后别忘进行检查

ifconfig eth0 –检查网卡配置信息
ping www.baidu.com —检查网络是否通畅

虚拟主机相关常见问题

01）虚拟主机中的系统日常不要关闭，挂起就好（建议）
02）如果显示虚拟主机打开有权限问题，将所有虚拟主机关闭后重新启动
03）可能影响虚拟主机网络畅通的因素—Windows主机中虚拟软件服务
windows +r –services.msc (共有5个服务)

远程软件说明

推荐xshell（毕竟便宜）

远程连接排查思路（去大保健之路）

01）路通不通（ping 目的）
原因：a.ip地址配置不正确
b.虚拟软件的网卡配置不正确
c.ping 10.0.0.200 -t ,重启一下网卡
2）防火墙程序
/etc/init.d/iptables stop – 关闭
/etc/init.d/iptables status –查看

3）确认大保健公司有没有营业（确保远程服务开启）
telnet 10.0.0.200 22
sshd 端口： 22
http 端口：80

连接测试成功：
[d:~]$ telnet 10.0.0.200 22

Connecting to 10.0.0.200:22…
Connection established.
To escape to local shell, press ‘Ctrl+Alt+]’.
SSH-2.0-OpenSSH_5.3

连接测试不成功
[d:~]$ telnet 10.0.0.200 22

Connecting to 10.0.0.200:22…
Could not connect to ‘10.0.0.200’ (port 22): Connection failed.
Type `help’ to learn how to use Xshell prompt.

/eth/init.d/sshd start   <--开启远程连接服务
/eth/init.d/sshd status  <--查看是否开启
