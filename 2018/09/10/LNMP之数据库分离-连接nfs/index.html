mysql分离以及nfs存储

mysql分离

部署安装mysql数据库软件

这就不啰嗦了

数据库数据备份

mysqldump -uroot -poldboy123 --all-databases >/tmp/bak.sql
ll /tmp/bak.sql -h
scp /tmp/bak.sql 172.16.1.51:/tmp/

迁移数据库备份信息

scp -rp /tmp/back.sql 172.16.1.51:/tmp

恢复数据库备份文件

mysql -uroot -poldboy123 </tmp/back.sql

修改代码配置文件

[root@web01 blog]# pwd
/application/nginx/html/blog
[root@web01 blog]# vim wp-config.php 
 /** MySQL主机 */
define('DB_HOST', '10.0.0.51');   --- 修改为数据库服务器地址

关闭web服务器上mysql服务

nfs存储

实现nginx服务与nfs服务建立联系

确认用户上传的数据保存在web服务器的哪个目录中
01) 利用浏览器页面信息获取数据存储路径
http://blog.etiantian.org/wp-content/uploads/2018/09/oldboy-300x267.jpg

02) 监控目录中的数据变化软件
inotifywait -mrq 

03) 利用find查找方法发现变化数据
find /站点目录  -mmin -3
将目录中的数据进行备份迁移
[root@web01 blog]# cd /application/nginx/html/blog/wp-content/uploads/

[root@web01 uploads]# mkdir /data_backup

[root@web01 uploads]# mv 2018/ /data_backup/
进行共享目录挂载操作
[root@web01 uploads]# mount -t nfs 172.16.1.31:/data ./
[root@web01 uploads]# df -h
Filesystem         Size  Used Avail Use% Mounted on
/dev/sda3           48G  4.0G   42G   9% /
tmpfs              238M     0  238M   0% /dev/shm
/dev/sda1          190M   35M  146M  19% /boot
172.16.1.31:/data   48G  1.8G   44G   4% /application/nginx-1.14/html/blog/wp-content/uploads
还原数据信息
cp -a /data_backup/2018/  /application/nginx/html/blog/wp-content/uploads/
进行测试访问
