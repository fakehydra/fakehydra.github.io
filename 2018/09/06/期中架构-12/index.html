规范化配置虚拟主机配置文件

命令如下
vim nginx.conf
：set nu --显示行号
sed -n '10,16p' nginx.conf >extra/www.conf
sed -n '17,22p' nginx.conf >extra/bbs.conf
sed -n '23,28p' nginx.conf >extra/blog.conf

vim nginx.conf
      worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;
    include       extra/www.conf;
    include       extra/bbs.conf;
    include       extra/blog.conf;
}

 server {
    listen       80;
    server_name  www.jinc.org www.jc.org jinc.org;
}

网站服务日志配置

错误日志

[root@web01 conf]# vim nginx.conf
worker_processes  1;
error_log  logs/error_jinc
补充日志错误级别:
debug    --- 调试级别  输出日志信息最全
info     --- 普通级别  输出提示信息 
notice   --- 注意级别  输出注意信息 
warn     --- 警告级别  输出一些无关紧要的错误信息 
error    --- 错误级别  有影响服务正常运行的错误了(重点关注)
crit     --- 严重错误  比错误级别更严重些
alert,   --- 更严重级别
emerg.   --- 崩溃级别   输出的日志信息较少

访问日志

访问日志配置方法:
log_format  main '$remote_addr - $remote_user [$time_local] "$request" '
                 '$status $body_bytes_sent "$http_referer" '
                 '"$http_user_agent" "$http_x_forwarded_for"';
log_format  oldboy  '$remote_addr "$request" $status $body_bytes_sent'

access_log  logs/access_default.log  main;
access_log  logs/access_oldboy.log  oldboy;

$remote_addr          10.0.0.1                --- 访问者源地址信息
- 
$remote_user             -                    --- nginx服务进行认证访问时, 显示认证用户信息
[$time_local]                                 --- 显示访问时间戳信息
"$request"             "GET / HTTP/1.1"           --- 请求行的内容
$status             200                     --- 显示状态码信息
                                               补充: 304状态,表示客户端本地有缓存信息
$body_bytes_sent    1498                    --- 响应数据的大小信息
"$http_referer"     "-"                     --- 显示通过什么网站超链接到本地网站上
"$http_user_agent"                          --- 客户端访问的浏览器信息
"$http_x_forwarded_for"                     --- ???

日志切割

1.编写日志切割脚本

mv access.log access.log.$(date +%F)
nginx -s reload
rm -rf $(find -mtime +7)

2.通过系统自带的logrotate

[root@web01 ~]# cat /etc/logrotate.d/
ConsoleKit  dracut      psacct      syslog      yum         
[root@web01 ~]# cat /etc/logrotate.d/syslog 
/var/log/cron
/var/log/maillog
/var/log/messages
/var/log/secure
/var/log/spooler
{
    sharedscripts
    postrotate
    /bin/kill -HUP `cat /var/run/syslogd.pid 2> /dev/null` 2> /dev/null || true
    endscript
}

nginx访问认证功能

先确认stub模块开启了
nginx -V 
在最后一行会有：configure arguments: --prefix=/application/nginx --user=www --group=www --with-http_ssl_module --with-http_stub_status_module
确定之后在配置文件中添加一个location

   location /status {
       stub_status on;
       access_log   off;
                         }
保存退出

创建密码认证文件并且授权

这里我们用的是httpd 中的htpasswd
先下载
yum install -y httpd-tools
然后创建：
htpasswd -bc /application/nginx/conf/htpasswd admin admin
    -c  Create a new file.
   创建一个新的密码文件
   -b  Use the password from the command line rather than prompting for it.
   采用免交互的方式输入用户的密码信息
修改权限：
chmod 400 1/application/nginx/conf/htpasswd
chowm www /application.nginx/conf/htpasswd
在浏览器测试：输入地址 www.jinc.org/status
会提示输入账号密码：admin admin 就行了
Linux测试：
[root@web01 conf]# curl -u admin:admin www.jinc.org

</head>

</html>
