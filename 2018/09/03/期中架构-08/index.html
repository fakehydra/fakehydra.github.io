免交互分发秘钥

批量分发秘钥实现脚本

vim batch_fenfa.sh
#!/bin/bash
#create key pair

rm -f /root/.ssh/id_dsa*
ssh-keygen -t dsa -f /root/.ssh/id_dsa -P "" -q

#fenfa key file
for ip in 8 31 41
do
  echo "============ fenfa key to host 172.16.1.$ip ============"
  sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub "-o StrictHostKeyChecking=no root@172.16.1.$ip"
  echo "==================== fenfa key end ====================="
  echo ""
done

给50台主机分发公钥,实现基于秘钥远程管理?

1. 分发公钥时需要输入密码信息        sshpass
2. 分发公钥时总有确认信息(yes/no)  需要查看ssh-copy-id
3. 远程主机ssh服务端口号,如果不是22端口如何分发

1) 需要进行密码交互
[root@m01 ~]# sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub 172.16.1.41
Now try logging into the machine, with "ssh '172.16.1.41'", and check in:

  .ssh/authorized_keys

to make sure we haven't added extra keys that you weren't expecting.

2) 需要输入确认信息(yes/no)
ssh-copy-id脚本中内容说明
ssh $1 "exec sh -c 'cd; umask 077; test -d .ssh || mkdir .ssh ; cat >> .ssh/authorized_keys && \
(test -x /sbin/restorecon && /sbin/restorecon .ssh .ssh/authorized_keys >/dev/null 2>&1
1) 利用ssh进行远程连接,对远程主机进行操作($1 表示是一个远程主机IP地址信息)
2) 先进入远程主机家目录中,并设置umask值为077
3) 判断是否存在.ssh目录,如果没有.ssh目录进行创建
4) 让远程主机生成公钥文件authorized_keys,并设置权限为600

[root@m01 ~]# sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub "172.16.1.41 -o StrictHostKeyChecking=no"

  .ssh/authorized_keys

to make sure we haven't added extra keys that you weren't expecting.

[root@m01 ~]# sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub "172.16.1.31 -p52113 -o  StrictHostKeyChecking=no"

脚本完善补充:实现免交互创建秘钥对
1)不要再问我私钥保存到什么位置了?
ssh-keygen -t dsa -f “/root/.ssh/id_dsa”

2) 私钥密码信息不要再问我了?
rm -f /root/.ssh/id_dsa*
ssh-keygen -t dsa -f "/root/.ssh/id_dsa" -N "" -q
echo "key pair create ok!!!"

-N new_passphrase
   Provides the new passphrase.
-P passphrase
   Provides the (old) passphrase.

ansible 批量管理软件介绍(运维开发)

1) Python语言是运维人员必会的语言!
   ansible是一个基于Python开发的自动化运维工具!
2) 其功能实现基于ssh远程连接服务
3) ansible 可以实现批量系统配置,批量软件部署,批量文件拷贝,批量运行命令等功能

ansible 软件特点概述(saltstack puppet)

1)不需要单独安装客户端(no agents),基于系统自带的sshd服务,sshd就相当于ansible的客户端
2)不需要服务端(no server)
3)需要依靠大量的模块实现批量管理
4)配置文件/etc/ansible/ansible.cfg

ansible软件的安装部署过程

管理端部署
第一个里程:    部署ssh+key 远程连接主机环境

第二个里程: 安装ansible软件程序
说明: ansible软件安装需要有epel源
yum install -y ansible
补充:将一个jobs能查看到的进程杀死,并扔掉
 yum install -y keepalived  --- ctrl+z 暂停 --- kill % 

/etc/ansible              --- ansible程序目录
/etc/ansible/ansible.cfg  --- ansible默认配置文件
/etc/ansible/hosts        --- ansible主机管理文件
/etc/ansible/roles        --- ansible角色目录
/usr/bin/ansible          --- ansible程序命令(执行一个模块)
/usr/bin/ansible-playbook --- ansible程序命令(执行一个剧本)

第三里程:配置编写主机管理文件

利用ansible实现批量管理方法

ansible批量管理命令语法
ansible 主机信息或模块信息  -m 模块名(命令)  -a "模块参数(参数信息)"

管理单个主机

[root@m01 ansible]# ansible 172.16.1.31 -m command -a 'hostname'
172.16.1.31 | SUCCESS | rc=0 >>
nfs01

[root@m01 ansible]# ansible 172.16.1.31 -m command -a 'hostnam'
172.16.1.31 | FAILED | rc=2 >>
[Errno 2] No such file or directory

管理多个主机

[root@m01 ansible]# ansible 172.16.1.31,172.16.1.41 -m command -a 'hostname'
172.16.1.31 | SUCCESS | rc=0 >>
nfs01

172.16.1.41 | SUCCESS | rc=0 >>
backup

管理所有主机

[root@m01 ansible]# ansible all -m command -a 'hostname'
172.16.1.41 | SUCCESS | rc=0 >>
backup

172.16.1.31 | SUCCESS | rc=0 >>
nfs01

补充说明: 不让分发公钥如何远程管理

1) 编写hosts文件,添加用户名称和密码信息
[oldboy02]
172.16.1.7 ansible_ssh_user=root ansible_ssh_pass=123456 ansible_port=52113

2) 执行ansible命令进行测试    
ansible oldboy02 -m command -a 'df -h'
172.16.1.7 | SUCCESS | rc=0 >>
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda3        48G  1.9G   44G   5% /
tmpfs           238M     0  238M   0% /dev/shm
/dev/sda1       190M   35M  146M  19% /boot

3)不将密码写入配置文件,临时远程管理  
[oldboy02]
172.16.1.7 ansible_ssh_user=root ansible_port=52113

在执行ansible远程连接管理命令后面加上一个 -k 参数,再输入密码即可实现
[root@m01 ansible]# ansible oldboy02 -m command -a 'df -h' -k
SSH password:
