免交互分发秘钥

批量分发秘钥实现脚本

vim batch_fenfa.sh
#!/bin/bash
#create key pair

rm -f /root/.ssh/id_dsa*
ssh-keygen -t dsa -f /root/.ssh/id_dsa -P "" -q

#fenfa key file
for ip in 8 31 41
do
  echo "============ fenfa key to host 172.16.1.$ip ============"
  sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub "-o StrictHostKeyChecking=no root@172.16.1.$ip"
  echo "==================== fenfa key end ====================="
  echo ""
done

给50台主机分发公钥,实现基于秘钥远程管理?

1. 分发公钥时需要输入密码信息        sshpass
2. 分发公钥时总有确认信息(yes/no)  需要查看ssh-copy-id
3. 远程主机ssh服务端口号,如果不是22端口如何分发

1) 需要进行密码交互
[root@m01 ~]# sshpass -p123456 ssh-copy-id -i /root/.ssh/id_dsa.pub 172.16.1.41
Now try logging into the machine, with "ssh '172.16.1.41'", and check in:

  .ssh/authorized_keys

to make sure we haven't added extra keys that you weren't expecting.

2) 需要输入确认信息(yes/no)
ssh-copy-id脚本中内容说明
ssh $1 "exec sh -c 'cd; umask 077; test -d .ssh || mkdir .ssh ; cat >> .ssh/authorized_keys && \
(test -x /sbin/restorecon && /sbin/restorecon .ssh .ssh/authorized_keys >/dev/null 2>&1
1) 利用ssh进行远程连接,对远程主机进行操作($1 表示是一个远程主机IP地址信息)
2) 先进入远程主机家目录中,并设置umask值为077
3) 判断是否存在.ssh目录,如果没有.ssh目录进行创建
4) 让远程主机生成公钥文件authorized_keys,并设置权限为600

ansible 软件特点概述(saltstack puppet)

1)不需要单独安装客户端(no agents),基于系统自带的sshd服务,sshd就相当于ansible的客户端
2)不需要服务端(no server)
3)需要依靠大量的模块实现批量管理
4)配置文件/etc/ansible/ansible.cfg

ansible批量管理软件的模块

command 模块–命令模块

功能说明
参数
chdir--先切换目录再执行操作
eg:

minute:                # Minute when the job should run ( 0-59, *, */2, etc )

day:                   # Day of the month the job should run ( 1-31, *, */2, etc )
month:                 # Month of the year the job should run ( 1-12, *, */2, etc )
weekday:               # Day of the week that the job should run ( 0-6 for Sunday-Saturday, *, etc )

job:                   # The command to execute or, if env is set, the value of environment variable. The command

 扩展说明：jinja2模板作用：
【 for循环语句jinja模板】
 为远程主机生成服务器列表，加入该列表info从192.168.13.201 web01.test.com 到192.168.13.211 web11.test.com 结束，
 如果手动添加就很不科学了，这里需要使用jinja2语法的for循环通过模板批量生成对应的配置文件，如下：

 ansible目录结构：
 #cd /etc/ansible/roles/test_hosts
 .
 ├── meta
 │   └── main.yml
 ├── tasks
 │   ├── file1.yml
 │   └── main.yml
 ├── templates
 │   └── test1.j2
 └── vars
     └── main.yml

各个目录下yml文件内容：
# cat tasks/main.yml 
- include: file1.yml

# cat tasks/file1.yml 
- name: ansible jinja2 template for hosts config
  template: src=test1.j2 dest=/etc/httpd/conf/httpd.conf.test

# cat templates/test1.j2 
  {% for id in range(201,212) %}
      192.168.13.{{ id }} web{{ "%02d" |format(id-200) }}.test.com
      {% endfor %}
  解释：
  {{ id }} 提取for循环中对应的变量id值
  "%02d"   调用的是python内置的字符串格式化输出（%d格式化整数）因为是01,02这种格式，所以是保留2位，故用02
  然后将结果通过管道符 “|” 传递给format 函数做二次处理。

如何实现配置定时任务

PS：定时任务执行的最小周期，是每分钟，或者间隔1分钟

测验练习题：每隔3分钟，执行时间同步任务 
8:00  ntpdate  ntp1.aliyun.com
8:03  ntpdate  ntp1.aliyun.com
8:06  ntpdate  ntp1.aliyun.com

[root@jinc ~]# crontab -l
*/3 * * * * /usr/sbin/ntpdate ntp1.aliyun.com



编写定时任务规范流程：
第一个里程：测试要执行的任务命令
ntpdate ntp1.aliyun.com

第二个里程：编写定时任务
# 时间同步注释信息
crontab -e（推荐）/ vim /var/spool/cron/root   -- crontab -e 编写定时任务信息时，有语法检查功能
*/3 * * * * ntpdate ntp1.aliyun.com

第三个里程：检查定时任务是否配置成功
1）检查定时任务是否已经配置上

2）检查定时任务是否有效果（每分钟测试）

第四个里程：进行定时任务异常排查
1）将定时任务里面的命令取出，在命令行手动再次执行一遍

2）看定时任务执行的过程（查看日志文件）    
说明：定时任务里面执行命令的时候，不会调取PATH环境变量信息
      定时任务里面的命令要用绝对路径执行
      一些内置命令在定时任务里面执行的时候，可以不用写绝对路径
